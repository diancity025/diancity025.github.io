00014                         ##### How to use this driver #####
00015   ==============================================================================
00016   [..]
00017     The UART HAL driver can be used as follows:
00018 
00019     (#) Declare a UART_HandleTypeDef handle structure (eg. UART_HandleTypeDef huart).
00020     (#) Initialize the UART low level resources by implementing the HAL_UART_MspInit() API:
00021         (##) Enable the USARTx interface clock.
00022         (##) UART pins configuration:
00023             (+++) Enable the clock for the UART GPIOs.
00024             (+++) Configure these UART pins (TX as alternate function pull-up, RX as alternate function Input).
00025         (##) NVIC configuration if you need to use interrupt process (HAL_UART_Transmit_IT()
00026              and HAL_UART_Receive_IT() APIs):
00027             (+++) Configure the USARTx interrupt priority.
00028             (+++) Enable the NVIC USART IRQ handle.
00029         (##) DMA Configuration if you need to use DMA process (HAL_UART_Transmit_DMA()
00030              and HAL_UART_Receive_DMA() APIs):
00031             (+++) Declare a DMA handle structure for the Tx/Rx channel.
00032             (+++) Enable the DMAx interface clock.
00033             (+++) Configure the declared DMA handle structure with the required
00034                   Tx/Rx parameters.
00035             (+++) Configure the DMA Tx/Rx channel.
00036             (+++) Associate the initialized DMA handle to the UART DMA Tx/Rx handle.
00037             (+++) Configure the priority and enable the NVIC for the transfer complete
00038                   interrupt on the DMA Tx/Rx channel.
00039             (+++) Configure the USARTx interrupt priority and enable the NVIC USART IRQ handle
00040                   (used for last byte sending completion detection in DMA non circular mode)
00041 
00042     (#) Program the Baud Rate, Word Length, Stop Bit, Parity, Hardware
00043         flow control and Mode(Receiver/Transmitter) in the huart Init structure.
00044 
00045     (#) For the UART asynchronous mode, initialize the UART registers by calling
00046         the HAL_UART_Init() API.
00047 
00048     (#) For the UART Half duplex mode, initialize the UART registers by calling
00049         the HAL_HalfDuplex_Init() API.
00050 
00051     (#) For the LIN mode, initialize the UART registers by calling the HAL_LIN_Init() API.
00052 
00053     (#) For the Multi-Processor mode, initialize the UART registers by calling
00054         the HAL_MultiProcessor_Init() API.
00055 
00056      [..]
00057        (@) The specific UART interrupts (Transmission complete interrupt,
00058             RXNE interrupt and Error Interrupts) will be managed using the macros
00059             __HAL_UART_ENABLE_IT() and __HAL_UART_DISABLE_IT() inside the transmit
00060             and receive process.
00061 
00062      [..]
00063        (@) These APIs (HAL_UART_Init() and HAL_HalfDuplex_Init()) configure also the
00064             low level Hardware GPIO, CLOCK, CORTEX...etc) by calling the customized
00065             HAL_UART_MspInit() API.

00068     ==================================
00069

00072     allows the user to configure dynamically the driver callbacks.
00073

00076     Function @ref HAL_UART_RegisterCallback() allows to register following callbacks:
00077     (+) TxHalfCpltCallback        : Tx Half Complete Callback.
00078     (+) TxCpltCallback            : Tx Complete Callback.
00079     (+) RxHalfCpltCallback        : Rx Half Complete Callback.
00080     (+) RxCpltCallback            : Rx Complete Callback.

00082     (+) AbortCpltCallback         : Abort Complete Callback.
00083     (+) AbortTransmitCpltCallback : Abort Transmit Complete Callback.

00085     (+) MspInitCallback           : UART MspInit.
00086     (+) MspDeInitCallback         : UART MspDeInit.

00088     and a pointer to the user callback function.
00089 
00090     [..]

00092     weak (surcharged) function.

00094     and the Callback ID.

00096     (+) TxHalfCpltCallback        : Tx Half Complete Callback.
00097     (+) TxCpltCallback            : Tx Complete Callback.
00098     (+) RxHalfCpltCallback        : Rx Half Complete Callback.
00099     (+) RxCpltCallback            : Rx Complete Callback.
00100     (+) ErrorCallback             : Error Callback.
00101     (+) AbortCpltCallback         : Abort Complete Callback.
00102     (+) AbortTransmitCpltCallback : Abort Transmit Complete Callback.
00103     (+) AbortReceiveCpltCallback  : Abort Receive Complete Callback.
00104     (+) MspInitCallback           : UART MspInit.
00105     (+) MspDeInitCallback         : UART MspDeInit.
00106 
00107     [..]
00108     By default, after the @ref HAL_UART_Init() and when the state is HAL_UART_STATE_RESET
00109     all callbacks are set to the corresponding weak (surcharged) functions:
00110     examples @ref HAL_UART_TxCpltCallback(), @ref HAL_UART_RxHalfCpltCallback().
00111     Exception done for MspInit and MspDeInit functions that are respectively
00112     reset to the legacy weak (surcharged) functions in the @ref HAL_UART_Init()
00113     and @ref HAL_UART_DeInit() only when these callbacks are null (not registered beforehand).
00114     If not, MspInit or MspDeInit are not null, the @ref HAL_UART_Init() and @ref HAL_UART_DeInit()
00115     keep and use the user MspInit/MspDeInit callbacks (registered beforehand).
00116

00118     Callbacks can be registered/unregistered in HAL_UART_STATE_READY state only.
00119     Exception done MspInit/MspDeInit that can be registered/unregistered
00120     in HAL_UART_STATE_READY or HAL_UART_STATE_RESET state, thus registered (user)
00121     MspInit/DeInit callbacks can be used during the Init/DeInit.
00122     In that case first register the MspInit/MspDeInit user callbacks
00123     using @ref HAL_UART_RegisterCallback() before calling @ref HAL_UART_DeInit()
00124     or @ref HAL_UART_Init() function.

00127     When The compilation define USE_HAL_UART_REGISTER_CALLBACKS is set to 0 or
00128     not defined, the callback registration feature is not available

01015 /**
01016   * @brief  Sends an amount of data in blocking mode.
01017   * @param  huart  Pointer to a UART_HandleTypeDef structure that contains
01018   *                the configuration information for the specified UART module.
01019   * @param  pData Pointer to data buffer
01020   * @param  Size Amount of data to be sent
01021   * @param  Timeout Timeout duration
01022   * @retval HAL status
01023   */
01024 HAL_StatusTypeDef HAL_UART_Transmit(UART_HandleTypeDef *huart, uint8_t *pData, uint16_t Size, uint32_t Timeout)
01025 {
01026   uint16_t *tmp;
01027   uint32_t tickstart = 0U;
01028 
01029   /* Check that a Tx process is not already ongoing */
01030   if (huart->gState == HAL_UART_STATE_READY)
01031   {
01032     if ((pData == NULL) || (Size == 0U))
01033     {
01034       return  HAL_ERROR;
01035     }
01036 
01037     /* Process Locked */
01038     __HAL_LOCK(huart);
01039 
01040     huart->ErrorCode = HAL_UART_ERROR_NONE;
01041     huart->gState = HAL_UART_STATE_BUSY_TX;
01042 
01043     /* Init tickstart for timeout managment */
01044     tickstart = HAL_GetTick();
01045 
01046     huart->TxXferSize = Size;
01047     huart->TxXferCount = Size;
01048     while (huart->TxXferCount > 0U)
01049     {
01050       huart->TxXferCount--;
01051       if (huart->Init.WordLength == UART_WORDLENGTH_9B)
01052       {
01053         if (UART_WaitOnFlagUntilTimeout(huart, UART_FLAG_TXE, RESET, tickstart, Timeout) != HAL_OK)
01054         {
01055           return HAL_TIMEOUT;
01056         }
01057         tmp = (uint16_t *) pData;
01058         huart->Instance->DR = (*tmp & (uint16_t)0x01FF);
01059         if (huart->Init.Parity == UART_PARITY_NONE)
01060         {
01061           pData += 2U;
01062         }
01063         else
01064         {
01065           pData += 1U;
01066         }
01067       }
01068       else
01069       {
01070         if (UART_WaitOnFlagUntilTimeout(huart, UART_FLAG_TXE, RESET, tickstart, Timeout) != HAL_OK)
01071         {
01072           return HAL_TIMEOUT;
01073         }
01074         huart->Instance->DR = (*pData++ & (uint8_t)0xFF);
01075       }
01076     }
01077 
01078     if (UART_WaitOnFlagUntilTimeout(huart, UART_FLAG_TC, RESET, tickstart, Timeout) != HAL_OK)
01079     {
01080       return HAL_TIMEOUT;
01081     }
01082 
01083     /* At end of Tx process, restore huart->gState to Ready */
01084     huart->gState = HAL_UART_STATE_READY;
01085 
01086     /* Process Unlocked */
01087     __HAL_UNLOCK(huart);
01088 
01089     return HAL_OK;
01090   }
01091   else
01092   {
01093     return HAL_BUSY;
01094   }
01095 }

01998   * @brief  This function handles UART interrupt request.
01999   * @param  huart  Pointer to a UART_HandleTypeDef structure that contains
02000   *                the configuration information for the specified UART module.
02001   * @retval None
02002   */
02003 void HAL_UART_IRQHandler(UART_HandleTypeDef *huart)
02004 {
02005   uint32_t isrflags   = READ_REG(huart->Instance->SR);
02006   uint32_t cr1its     = READ_REG(huart->Instance->CR1);
02007   uint32_t cr3its     = READ_REG(huart->Instance->CR3);
02008   uint32_t errorflags = 0x00U;
02009   uint32_t dmarequest = 0x00U;
02010 
02011   /* If no error occurs */
02012   errorflags = (isrflags & (uint32_t)(USART_SR_PE | USART_SR_FE | USART_SR_ORE | USART_SR_NE));
02013   if (errorflags == RESET)
02014   {
02015     /* UART in mode Receiver -------------------------------------------------*/
02016     if (((isrflags & USART_SR_RXNE) != RESET) && ((cr1its & USART_CR1_RXNEIE) != RESET))
02017     {
02018       UART_Receive_IT(huart);
02019       return;
02020     }
02021   }
02022 
02023   /* If some errors occur */
02024   if ((errorflags != RESET) && (((cr3its & USART_CR3_EIE) != RESET) || ((cr1its & (USART_CR1_RXNEIE | USART_CR1_PEIE)) != RESET)))
02025   {
02026     /* UART parity error interrupt occurred ----------------------------------*/
02027     if (((isrflags & USART_SR_PE) != RESET) && ((cr1its & USART_CR1_PEIE) != RESET))
02028     {
02029       huart->ErrorCode |= HAL_UART_ERROR_PE;
02030     }
02031 
02032     /* UART noise error interrupt occurred -----------------------------------*/
02033     if (((isrflags & USART_SR_NE) != RESET) && ((cr3its & USART_CR3_EIE) != RESET))
02034     {
02035       huart->ErrorCode |= HAL_UART_ERROR_NE;
02036     }
02037 
02038     /* UART frame error interrupt occurred -----------------------------------*/
02039     if (((isrflags & USART_SR_FE) != RESET) && ((cr3its & USART_CR3_EIE) != RESET))
02040     {
02041       huart->ErrorCode |= HAL_UART_ERROR_FE;
02042     }
02043 
02044     /* UART Over-Run interrupt occurred --------------------------------------*/
02045     if (((isrflags & USART_SR_ORE) != RESET) && ((cr3its & USART_CR3_EIE) != RESET))
02046     {
02047       huart->ErrorCode |= HAL_UART_ERROR_ORE;
02048     }
02049 
02050     /* Call UART Error Call back function if need be --------------------------*/
02051     if (huart->ErrorCode != HAL_UART_ERROR_NONE)
02052     {
02053       /* UART in mode Receiver -----------------------------------------------*/
02054       if (((isrflags & USART_SR_RXNE) != RESET) && ((cr1its & USART_CR1_RXNEIE) != RESET))
02055       {
02056         UART_Receive_IT(huart);
02057       }
02058 
02059       /* If Overrun error occurs, or if any error occurs in DMA mode reception,
02060          consider error as blocking */
02061       dmarequest = HAL_IS_BIT_SET(huart->Instance->CR3, USART_CR3_DMAR);
02062       if (((huart->ErrorCode & HAL_UART_ERROR_ORE) != RESET) || dmarequest)
02063       {
02064         /* Blocking error : transfer is aborted
02065            Set the UART state ready to be able to start again the process,
02066            Disable Rx Interrupts, and disable Rx DMA request, if ongoing */
02067         UART_EndRxTransfer(huart);
02068 
02069         /* Disable the UART DMA Rx request if enabled */
02070         if (HAL_IS_BIT_SET(huart->Instance->CR3, USART_CR3_DMAR))
02071         {
02072           CLEAR_BIT(huart->Instance->CR3, USART_CR3_DMAR);
02073 
02074           /* Abort the UART DMA Rx channel */
02075           if (huart->hdmarx != NULL)
02076           {
02077             /* Set the UART DMA Abort callback :
02078                will lead to call HAL_UART_ErrorCallback() at end of DMA abort procedure */
02079             huart->hdmarx->XferAbortCallback = UART_DMAAbortOnError;
02080             if (HAL_DMA_Abort_IT(huart->hdmarx) != HAL_OK)
02081             {
02082               /* Call Directly XferAbortCallback function in case of error */
02083               huart->hdmarx->XferAbortCallback(huart->hdmarx);
02084             }
02085           }
02086           else
02087           {
02088             /* Call user error callback */
02089 #if (USE_HAL_UART_REGISTER_CALLBACKS == 1)
02090             /*Call registered error callback*/
02091             huart->ErrorCallback(huart);
02092 #else
02093             /*Call legacy weak error callback*/
02094             HAL_UART_ErrorCallback(huart);
02095 #endif /* USE_HAL_UART_REGISTER_CALLBACKS */
02096           }
02097         }
02098         else
02099         {
02100           /* Call user error callback */
02101 #if (USE_HAL_UART_REGISTER_CALLBACKS == 1)
02102           /*Call registered error callback*/
02103           huart->ErrorCallback(huart);
02104 #else
02105           /*Call legacy weak error callback*/
02106           HAL_UART_ErrorCallback(huart);
02107 #endif /* USE_HAL_UART_REGISTER_CALLBACKS */
02108         }
02109       }
02110       else
02111       {
02112         /* Non Blocking error : transfer could go on.
02113            Error is notified to user through user error callback */
02114 #if (USE_HAL_UART_REGISTER_CALLBACKS == 1)
02115         /*Call registered error callback*/
02116         huart->ErrorCallback(huart);
02117 #else
02118         /*Call legacy weak error callback*/
02119         HAL_UART_ErrorCallback(huart);
02120 #endif /* USE_HAL_UART_REGISTER_CALLBACKS */
02121 
02122         huart->ErrorCode = HAL_UART_ERROR_NONE;
02123       }
02124     }
02125     return;
02126   } /* End if some error occurs */
02127 
02128   /* UART in mode Transmitter ------------------------------------------------*/
02129   if (((isrflags & USART_SR_TXE) != RESET) && ((cr1its & USART_CR1_TXEIE) != RESET))
02130   {
02131     UART_Transmit_IT(huart);
02132     return;
02133   }
02134 
02135   /* UART in mode Transmitter end --------------------------------------------*/
02136   if (((isrflags & USART_SR_TC) != RESET) && ((cr1its & USART_CR1_TCIE) != RESET))
02137   {
02138     UART_EndTransmit_IT(huart);
02139     return;
02140   }
02141 }
