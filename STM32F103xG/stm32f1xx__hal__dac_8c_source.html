00015   ==============================================================================
00016                       ##### DAC Peripheral features #####
00017   ==============================================================================
00018     [..]
00019       *** DAC Channels ***
00020       ====================
00021     [..]
00022     STM32F1 devices integrate two 12-bit Digital Analog Converters
00023 
00024     The 2 converters (i.e. channel1 & channel2)
00025     can be used independently or simultaneously (dual mode):
00026       (#) DAC channel1 with DAC_OUT1 (PA4) as output or connected to on-chip
00027           peripherals (ex. timers).
00028       (#) DAC channel2 with DAC_OUT2 (PA5) as output or connected to on-chip
00029           peripherals (ex. timers).
00030 
00031       *** DAC Triggers ***
00032       ====================
00033     [..]
00034     Digital to Analog conversion can be non-triggered using DAC_TRIGGER_NONE
00035     and DAC_OUT1/DAC_OUT2 is available once writing to DHRx register.
00036     [..]
00037     Digital to Analog conversion can be triggered by:
00038       (#) External event: EXTI Line 9 (any GPIOx_PIN_9) using DAC_TRIGGER_EXT_IT9.
00039           The used pin (GPIOx_PIN_9) must be configured in input mode.
00040 
00041       (#) Timers TRGO: TIM2, TIM4, TIM6, TIM7
00042           For STM32F10x connectivity line devices and STM32F100x devices: TIM3
00043           For STM32F10x high-density and XL-density devices: TIM8
00044           For STM32F100x high-density value line devices: TIM15 as
00045           replacement of TIM5.
00046           (DAC_TRIGGER_T2_TRGO, DAC_TRIGGER_T4_TRGO...)
00047   
00048       (#) Software using DAC_TRIGGER_SOFTWARE
00049 
00050       *** DAC Buffer mode feature ***
00051       ===============================
00052       [..]
00053       Each DAC channel integrates an output buffer that can be used to
00054       reduce the output impedance, and to drive external loads directly
00055       without having to add an external operational amplifier.
00056       To enable, the output buffer use
00057       sConfig.DAC_OutputBuffer = DAC_OUTPUTBUFFER_ENABLE;
00058       [..]
00059       (@) Refer to the device datasheet for more details about output
00060           impedance value with and without output buffer.
00061 
00062       *** DAC connect feature ***
00063       ===============================
00064       [..]
00065       Each DAC channel can be connected internally.
00066       To connect, use
00067       sConfig.DAC_ConnectOnChipPeripheral = DAC_CHIPCONNECT_ENABLE;
00068 
00069       *** GPIO configurations guidelines ***
00070       =====================
00071       [..]
00072       When a DAC channel is used (ex channel1 on PA4) and the other is not
00073       (ex channel2 on PA5 is configured in Analog and disabled).
00074       Channel1 may disturb channel2 as coupling effect.
00075       Note that there is no coupling on channel2 as soon as channel2 is turned on.
00076       Coupling on adjacent channel could be avoided as follows:
00077       when unused PA5 is configured as INPUT PULL-UP or DOWN.
00078       PA5 is configured in ANALOG just before it is turned on.
00079 
00080        *** DAC wave generation feature ***
00081        ===================================
00082        [..]
00083        Both DAC channels can be used to generate
00084          (#) Noise wave
00085          (#) Triangle wave
00086 
00087        *** DAC data format ***
00088        =======================
00089        [..]
00090        The DAC data format can be:
00091          (#) 8-bit right alignment using DAC_ALIGN_8B_R
00092          (#) 12-bit left alignment using DAC_ALIGN_12B_L
00093          (#) 12-bit right alignment using DAC_ALIGN_12B_R
00094 
00095        *** DAC data value to voltage correspondence ***
00096        ================================================
00097        [..]
00098        The analog output voltage on each DAC channel pin is determined
00099        by the following equation:
00100        [..]
00101        DAC_OUTx = VREF+ * DOR / 4095
00102        (+) with  DOR is the Data Output Register
00103        [..]
00104           VEF+ is the input voltage reference (refer to the device datasheet)
00105        [..]
00106         e.g. To set DAC_OUT1 to 0.7V, use
00107        (+) Assuming that VREF+ = 3.3V, DAC_OUT1 = (3.3 * 868) / 4095 = 0.7V
00108 
00109        *** DMA requests ***
00110        =====================
00111        [..]
00112        A DMA request can be generated when an external trigger (but not a software trigger)
00113        occurs if DMA1 requests are enabled using HAL_DAC_Start_DMA().
00114        DMA1 requests are mapped as following:
00115       (#) DAC channel1 mapped on DMA1 channel3
00116           for STM32F100x low-density, medium-density, high-density with DAC
00117           DMA remap:
00118       (#) DAC channel2 mapped on DMA2 channel3
00119           for STM32F100x high-density without DAC DMA remap and other  
00120           STM32F1 devices
00121 
00122      [..]
00123     (@) For Dual mode and specific signal (Triangle and noise) generation please
00124         refer to Extended Features Driver description

00143      *** DMA mode IO operation ***
00144      ==============================
00145      [..]
00146        (+) Start the DAC peripheral using HAL_DAC_Start_DMA(), at this stage the user specify the length
00147            of data to be transferred at each end of conversion
00148            First issued trigger will start the conversion of the value previously set by HAL_DAC_SetValue().
00149        (+) At the middle of data transfer HAL_DAC_ConvHalfCpltCallbackCh1() or HAL_DACEx_ConvHalfCpltCallbackCh2()
00150            function is executed and user can add his own code by customization of function pointer
00151            HAL_DAC_ConvHalfCpltCallbackCh1() or HAL_DACEx_ConvHalfCpltCallbackCh2()
00152        (+) At The end of data transfer HAL_DAC_ConvCpltCallbackCh1() or HAL_DACEx_ConvHalfCpltCallbackCh2()
00153            function is executed and user can add his own code by customization of function pointer
00154            HAL_DAC_ConvCpltCallbackCh1() or HAL_DACEx_ConvHalfCpltCallbackCh2()
00155        (+) In case of transfer Error, HAL_DAC_ErrorCallbackCh1() function is executed and user can
00156             add his own code by customization of function pointer HAL_DAC_ErrorCallbackCh1
00157        (+) For STM32F100x devices with specific feature: DMA underrun.             
00158            In case of DMA underrun, DAC interruption triggers and execute internal function HAL_DAC_IRQHandler.
00159            HAL_DAC_DMAUnderrunCallbackCh1() or HAL_DACEx_DMAUnderrunCallbackCh2()
00160            function is executed and user can add his own code by customization of function pointer
00161            HAL_DAC_DMAUnderrunCallbackCh1() or HAL_DACEx_DMAUnderrunCallbackCh2() and
00162            add his own code by customization of function pointer HAL_DAC_ErrorCallbackCh1()
00163        (+) Stop the DAC peripheral using HAL_DAC_Stop_DMA()

00165     *** Callback registration ***
00166     =============================================
00167     [..]
00168       The compilation define  USE_HAL_DAC_REGISTER_CALLBACKS when set to 1
00169       allows the user to configure dynamically the driver callbacks.
00170 
00171     Use Functions @ref HAL_DAC_RegisterCallback() to register a user callback,
00172       it allows to register following callbacks:
00173       (+) ConvCpltCallbackCh1     : callback when a half transfer is completed on Ch1.
00174       (+) ConvHalfCpltCallbackCh1 : callback when a transfer is completed on Ch1.
00175       (+) ErrorCallbackCh1        : callback when an error occurs on Ch1.
00176       (+) DMAUnderrunCallbackCh1  : callback when an underrun error occurs on Ch1.
00177       (+) ConvCpltCallbackCh2     : callback when a half transfer is completed on Ch2.
00178       (+) ConvHalfCpltCallbackCh2 : callback when a transfer is completed on Ch2.
00179       (+) ErrorCallbackCh2        : callback when an error occurs on Ch2.
00180       (+) DMAUnderrunCallbackCh2  : callback when an underrun error occurs on Ch2.
00181       (+) MspInitCallback         : DAC MspInit.
00182       (+) MspDeInitCallback       : DAC MspdeInit.
00183       This function takes as parameters the HAL peripheral handle, the Callback ID
00184       and a pointer to the user callback function.
00185 
00186     Use function @ref HAL_DAC_UnRegisterCallback() to reset a callback to the default
00187       weak (surcharged) function. It allows to reset following callbacks:
00188       (+) ConvCpltCallbackCh1     : callback when a half transfer is completed on Ch1.
00189       (+) ConvHalfCpltCallbackCh1 : callback when a transfer is completed on Ch1.
00190       (+) ErrorCallbackCh1        : callback when an error occurs on Ch1.
00191       (+) DMAUnderrunCallbackCh1  : callback when an underrun error occurs on Ch1.
00192       (+) ConvCpltCallbackCh2     : callback when a half transfer is completed on Ch2.
00193       (+) ConvHalfCpltCallbackCh2 : callback when a transfer is completed on Ch2.
00194       (+) ErrorCallbackCh2        : callback when an error occurs on Ch2.
00195       (+) DMAUnderrunCallbackCh2  : callback when an underrun error occurs on Ch2.
00196       (+) MspInitCallback         : DAC MspInit.
00197       (+) MspDeInitCallback       : DAC MspdeInit.
00198       (+) All Callbacks
00199       This function) takes as parameters the HAL peripheral handle and the Callback ID.
00200 
00201       By default, after the @ref HAL_DAC_Init and if the state is HAL_DAC_STATE_RESET
00202       all callbacks are reset to the corresponding legacy weak (surcharged) functions.
00203       Exception done for MspInit and MspDeInit callbacks that are respectively
00204       reset to the legacy weak (surcharged) functions in the @ref HAL_DAC_Init
00205       and @ref  HAL_DAC_DeInit only when these callbacks are null (not registered beforehand).
00206       If not, MspInit or MspDeInit are not null, the @ref HAL_DAC_Init and @ref HAL_DAC_DeInit
00207       keep and use the user MspInit/MspDeInit callbacks (registered beforehand)
00208 
00209       Callbacks can be registered/unregistered in READY state only.
00210       Exception done for MspInit/MspDeInit callbacks that can be registered/unregistered
00211       in READY or RESET state, thus registered (user) MspInit/DeInit callbacks can be used
00212       during the Init/DeInit.
00213       In that case first register the MspInit/MspDeInit user callbacks
00214       using @ref HAL_DAC_RegisterCallback before calling @ref HAL_DAC_DeInit
00215       or @ref HAL_DAC_Init function.
00216 
00217       When The compilation define USE_HAL_DAC_REGISTER_CALLBACKS is set to 0 or
00218       not defined, the callback registering feature is not available
00219       and weak (surcharged) callbacks are used.
